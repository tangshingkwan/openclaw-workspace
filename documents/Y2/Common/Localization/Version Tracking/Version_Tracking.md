### 2.     G ERMANY

To communicate with EFSTA's EFR Taxbox, you need to install version 1.6.24 or higher of the Cegid

CPOS driver "CPOS.TaxBox.DE.EFSTA".

Earlier versions of the Cegid CPOS driver "CPOS.TaxBox.DE.EFSTA" are no longer supported by Cegid.


**2.1.1.   CPOS.TaxBox.DE.EFSTA  Version 1.7.25 on 01/22/2024**

Driver settings for Taxbox sharing (mobile systems)

Location ID parameter modified:  Possibility of leaving it blank. If this is the case, the CPOS driver will

use the store code associated with the cash register to communicate with the EFSTA EFR.

Modification of Terminal ID parameter. Possibility of leaving it blank. In this case, the CPOS driver

will use the cash register code for dialogue with EFSTA's EFR.

This upgrade enables shared access to the TaxBox via InStoreHub.

For Live Store, a version of InStoreHub 1.3.310 or higher is required.

Not supported by mPOS.


**2.1.2.   CPOS.TaxBox.DE.EFSTA  Version 1.7.26 on 07/11/2024**

Correction to the ERF availability test

During the first EFR availability test, the test was systematically performed on the localhost IP address

instead of the IP address defined in the driver settings. This only returned an error when the EFR was

not installed on the cash register.


**2.1.3.   CPOS.TaxBox.DE.EFSTA Version 1.7.28 on 10/08/2024**

Corrected RECEIPTID verb return error (Live Store only)

Regression introduced with version 1.7.26. The driver would return an error if the CPOS RECEIPTID

verb did not return a value. This error only occurred when using driver version 1.7.26 with Live Store.

CPOSJournalViewer.exe version 1.0.0.10 dated 21/08/2018 is used to view files containing the

electronic journal.

First version


**7.1.2.   Cegid.CPOS.my.Edicom Version 2.1.7 on 10/10/2024**

Allows B2C transactions to be validated in Y2 without waiting for Edicom's response.


**7.1.3.   Cegid.CPOS.my.Edicom Version 2.2.9 on 10/31/2024**

Added missing tags

The tags <Taxable_amount> and <Tax_exemption_reason> under the tag <TAX_LINE>are added

VAT rate for the row in question is equal to 0.

The tags <Additional_ID_type> and <Additional_ID_value> are added by default for a customer.

Rejection of returns with multiple rows

Return transactions with several lines of the same original receipt are no longer rejected.


**7.1.4.   Cegid.CPOS.Edicom Version 2.3.11 on 11/15/2024**

Date and time in UTC time zone

Documents are transmitted with date and time set in the UTC time zone, instead of in the cash

register's local time zone. It is therefore important that Windows is set to the correct time zone on

the cash register.

Improvement of the return with receipt process

The “IRBM ID” is no longer required if the return receipt is associated with an original receipt.

Ability to set return document type

It is now possible to set the type of return document to replace the default “04”.

Setting Financial Items to be excluded from transmission to the tax authorities

Implementation of a setting to define a list of Financial Items for which the transaction receipt will

not be sent to the tax authorities, provided that the receipt contains only the specified items and

that these items have a VAT rate of 0%.
